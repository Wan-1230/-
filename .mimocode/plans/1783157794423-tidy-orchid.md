# 简历优化功能实现计划

## 功能描述

根据用户粘贴的 JD（职位描述）和简历内容，调用 MiMo LLM 分析 JD 要求，生成优化后的简历，并标注修改建议。

## 实现方案

### 1. 新建前端页面 `src/pages/ResumePage.tsx`

**布局设计：**
- 左右分栏（移动端上下排列）
- 左侧：JD 输入框（textarea）+ 简历输入框（textarea）+ "开始优化" 按钮
- 右侧：优化结果展示区（Markdown 渲染 + 复制按钮）
- 底部：加载动画（LLM 生成中）

**交互流程：**
1. 用户粘贴 JD
2. 用户粘贴简历原文
3. 点击"开始优化"
4. 调用后端 `/api/resume/optimize` 接口
5. 流式返回优化结果，实时展示
6. 支持一键复制优化后的简历

### 2. 新建前端 API `src/lib/resumeApi.ts`

提供 `optimizeResume(jd, resume)` 函数，调用后端流式接口，返回优化结果。

### 3. 新增后端接口 `backend/server.js`

添加 `POST /api/resume/optimize` 端点：
- 接收 `{ jd, resume }` 请求体
- 构造专门的 Prompt，将 JD 和简历交给 LLM 分析
- 流式返回优化结果（SSE）

**Prompt 设计要点：**
- 先分析 JD 的核心要求（技能、经验、职责）
- 对比简历当前内容，找出差距
- 生成优化后的简历（保持原格式，增强匹配度）
- 给出具体修改建议和说明

### 4. 更新路由 `src/App.tsx`

添加 `/resume` 路由指向 `ResumePage`。

### 5. 更新导航 `src/components/Header.tsx`

在 navItems 中添加"简历优化"入口。

## 验证方式

1. 启动后端 `cd backend && npm start`
2. 启动前端 `npm run dev`
3. 访问 `/resume` 页面
4. 粘贴一段 JD 和简历，点击优化
5. 确认流式返回结果正常显示
